# Аудит сайта РегионДорСтрой

Экспертный разбор по 6 измерениям (производительность, SEO, UX, конверсия,
доступность, безопасность/право) — аналог того, что выдают WebScore.ai /
Roast My Website / WhyBounce / Capian, но с учётом бизнес-логики
(дорожно-строительные услуги + аренда техники + продажа материалов в Орле).

Разбор сделан по исходной вёрстке (Claude Design `РегионДорСтрой.dc.html`) и
собранному `index.html`.

**Итерация 2.** Дизайн обновлён — применена бóльшая часть правок из итерации 1.

**Оценка: 7.5/10** (было 6/10). Прорыв по адаптивности, доступности и UX
каталога. Главный остаток — сайт всё ещё **теряет лиды** (формы без бэкенда) и
не имеет реальных фото/доказательств.

## ✅ Исправлено в этой итерации

- **Адаптивность** — добавлены `@media` (1024/900/768/640), сетки корректно
  схлопываются: техника/материалы 4→2→1, услуги 3→2→1, этапы 5→3→1.
- **Мобильное меню** — появилась кнопка-бургер + выпадающее меню; десктоп-навигация
  и телефон скрываются <900px. Раньше навигации на телефоне не было вообще.
- **«Корзина» → «Запросить расчёт»** — тупиковый счётчик убран. Кнопка на карточке
  материала открывает модалку с предзаполненным названием материала и уходит в ту
  же воронку заявки.
- **Доступность** — `focus-visible` обводка, `aria-label` на иконках (эмодзи
  помечены `aria-hidden`), `aria-label` на полях форм, контраст топбара поднят
  (`#b8b8b8`/`#cfcfcf` вместо `#9a9a9a`).
- **Согласие 152-ФЗ** — добавлен обязательный чекбокс согласия в обе формы.
- **Производительность** — `logo.png` (192 КБ) → `logo-opt.png` (91 КБ, 226×300);
  веса шрифтов обрезаны до используемых (Oswald 600/700, Manrope 400/600/700/800).
- **Баг ховера услуг** — текст на чёрном фоне теперь читаемый (h3 белый, p серый).

## 🔴 Критично — всё ещё открыто

### 1. Формы не отправляются — лиды теряются
Обе формы (заявка + модалка звонок/расчёт) по сабмиту просто показывают «Спасибо»
на клиенте. **Никуда не уходят** — ни на почту, ни в CRM, ни в Telegram.
→ Главный убийца конверсии. Подключить обработчик: Formspree / своя ручка на бэке
/ бот в Telegram / интеграция с Reply24. Минимум — отправка на
`Rdstroiy@yandex.ru` + дубль в Telegram. Расчёт материала должен передавать
выбранный материал.

### 2. Нет реальных фото
Все визуалы (hero, 8 единиц техники, 4 материала) — пустые плейсхолдеры
(`Фото: …`). Дорожник продаётся **видимым доказательством**: техника, объекты
до/после, бригады.
→ Залить реальные фото парка и выполненных объектов.

---

## 🟠 Важно

### 3. SEO — частично
В `index.html` есть `<title>`, `description`, `lang="ru"`. **Нет** `LocalBusiness`
schema.org, Open Graph, favicon, `robots.txt`, `sitemap.xml`. Исходный `.dc.html`
вообще без `<title>`/meta (добавляются только при сборке).
→ Критично для локального SEO: `LocalBusiness` (адрес, телефон, часы, гео) →
Яндекс/Google Карты. OG-теги для красивых ссылок в WhatsApp/Telegram (основные
каналы). Favicon.

### 4. Аналитики нет
Ни Яндекс.Метрики, ни GA. Ничего не измеряется.
→ Поставить **Яндекс.Метрику** с вебвизором и картой кликов/скроллинга — бесплатно
закрывает то, ради чего в списке был VWO (heatmaps, rage-clicks, записи сессий).

---

## 🟡 Доверие, право

### 5. Нет доказательств и соцподтверждения
Нет отзывов, кейсов до/после, лицензий/допусков, гарантий с конкретикой. Цифры
«5–10 заказов в неделю» и «100% качество» — слабые.
→ Блок выполненных объектов (фото + м²/адрес), отзывы, реальные цифры (лет на
рынке, км дорог, объектов сдано).

### 6. Контакты без карты
Адрес только текстом → нужна встроенная **Яндекс.Карта**.

### 7. Право (152-ФЗ) — наполовину
Чекбокс согласия есть, но **нет страницы «Политика конфиденциальности»** (ссылка в
тексте ведёт в никуда) и **нет реквизитов** (ИНН/ОГРН, юр. адрес) в подвале.
→ Добавить страницу политики + реквизиты ООО.
