# Аудит сайта РегионДорСтрой

Экспертный разбор по 6 измерениям (производительность, SEO, UX, конверсия,
доступность, безопасность/право) — аналог того, что выдают WebScore.ai /
Roast My Website / WhyBounce / Capian, но с учётом бизнес-логики
(дорожно-строительные услуги + аренда техники + продажа материалов в Орле).

Разбор сделан по исходной вёрстке (Claude Design `РегионДорСтрой.dc.html`) и
собранному `index.html`.

**Общая оценка: 6/10.** Дизайн сильный (фирстиль, типографика Oswald+Manrope,
анимации, структура секций). Но как **инструмент привлечения заявок** сайт
сейчас дырявый: не работает на мобильных, теряет лиды (формы без бэкенда), нет
реальных фото и доказательств, не измеряется.

---

## 🔴 Критично (чинить в первую очередь)

### 1. Сайт не адаптивен — ломается на телефоне
Все сетки фиксированные: услуги `repeat(3,1fr)`, техника/материалы `repeat(4,1fr)`,
этапы `repeat(5,1fr)`, hero/«о компании»/контакты — 2 колонки. **Ни одного
`@media`.** На телефоне карточки сплющиваются до нечитаемых, `overflow-x:hidden`
на `body` просто обрезает вылезающее.
Меню в шапке (`flex; gap:28px` + лого + телефон + 2 кнопки в строке 78px)
переполняется. В JS есть `toggleMenu`/`state.menu`, но **кнопки-гамбургера в
вёрстке нет** → на мобильном нет навигации.
→ **Для локального бизнеса 60–70% трафика мобильные. Это потеря большинства
посетителей.** Добавить media-queries: 1 колонка на ≤640px, 2 на ≤960px,
бургер-меню, уменьшить hero h1 (сейчас фикс 62px).

### 2. Формы не отправляются — лиды теряются
Обе формы (заявка + «заказать звонок») по сабмиту просто показывают «Спасибо»
на клиенте. **Никуда не уходят** — ни на почту, ни в CRM, ни в Telegram.
→ Главный убийца конверсии. Подключить обработчик: Formspree / своя ручка на
бэке / бот в Telegram / интеграция с Reply24. Минимум — отправка на
`Rdstroiy@yandex.ru` + дубль в Telegram.

### 3. «Корзина» для стройматериалов — тупиковая
Кнопка «В корзину» только увеличивает счётчик. **Нет ни просмотра корзины, ни
оформления.** Для оптовых сыпучих (щебень/песок «от N ₽ за тонну/м³») модель
корзины вообще странная — клиенту нужен расчёт объёма и доставки, а не e-commerce.
→ Заменить «В корзину» на «Запросить расчёт» → та же форма заявки с
предзаполненным материалом. Счётчик-корзину убрать или довести до конца.

### 4. Нет реальных фото
Все визуалы (hero, 8 единиц техники, 4 материала) — пустые плейсхолдеры
(`Фото: …`). Дорожник продаёт **видимым доказательством**: техника, объекты
до/после, бригады.
→ Залить реальные фото парка и выполненных объектов. Без них доверие и
конверсия низкие.

## 🟠 Важно

### 5. SEO — почти пусто
Исходник `.dc.html`: **нет `<title>`, `description`, `lang`, Open Graph,
favicon, микроразметки.** (В `index.html` я добавил `<title>`, `description`,
`lang="ru"` — остальное нужно доделать.)
→ Критично для локального SEO: **schema.org `LocalBusiness`** (адрес, телефон,
часы, гео) — попадание в Яндекс/Google Карты. Плюс OG-теги для красивых ссылок в
мессенджерах (WhatsApp/Telegram — основные каналы тут), favicon, `robots.txt`,
`sitemap.xml`, человекочитаемый title с гео («…в Орле»).

### 6. Аналитики нет
Ни Яндекс.Метрики, ни GA. **Ничего не измеряется** — нельзя понять, откуда
заявки и где отваливаются.
→ Поставить **Яндекс.Метрику** с вебвизором и картой кликов/скроллинга. Это
бесплатно закрывает то, ради чего в списке был VWO (heatmaps, rage-clicks,
записи сессий).

### 7. Доступность (a11y)
- Поля форм без `<label>` — только `placeholder` (исчезает при вводе, плохо для
  скринридеров и автозаполнения).
- Иконки — эмодзи (📞 ✉ 📍 🛒 ⚙) без `aria-label`.
- Контраст: серый `#9a9a9a` на чёрном `#0a0a0a` в топбаре — ниже нормы WCAG AA.
- Нет видимого focus-стиля кроме смены цвета рамки.
→ Добавить `<label>`/`aria-label`, поднять контраст, focus-ring.

### 8. Производительность
- **`logo.png` — 192 КБ** при показе 84px и 54px. Жуть.
  → Пережать до <15 КБ или сделать SVG.
- Google Fonts: **2 семейства × 8 начертаний** — грузится больше, чем
  используется, рендер-блокинг.
  → Оставить нужные веса (Oswald 600/700, Manrope 400/700/800), `display=swap`
  уже есть.
- Вся стилизация — инлайн (49 КБ HTML). Работает, но раздувает документ и мешает
  кешированию. Не срочно.

## 🟡 Доверие, конверсия, право

### 9. Нет доказательств и социального подтверждения
Нет отзывов, кейсов до/после, логотипов клиентов, лицензий/допусков, гарантий с
конкретикой. Цифры «5–10 заказов в неделю» и «100% качество» — слабые и
неубедительные.
→ Добавить блок выполненных объектов (фото + м²/адрес), отзывы, реальные цифры
(лет на рынке, км дорог, объектов сдано).

### 10. Контакты без карты
Адрес только текстом. Для локального бизнеса нужна **встроенная Яндекс.Карта**.

### 11. Право (152-ФЗ)
Под формами текст про обработку перс. данных, но **нет ссылки на политику
конфиденциальности и нет чекбокса согласия**. Для ООО — риск.
→ Добавить страницу «Политика конфиденциальности», чекбокс согласия, реквизиты
(ИНН/ОГРН, юр. адрес) в подвал.

### 12. Баг ховера на карточках услуг
В исходнике при наведении фон карточки → чёрный, но текст остаётся тёмным
(`h3 color:inherit`, `p #6a6a6a`) → **нечитаемо**. В `index.html` я поправил
(текст белеет на ховере) — проверить, что это желаемое поведение.

## Про сервисы из исходного списка

- **WebScore.ai / Roast My Website / WhyBounce / Capian** — этот разбор покрывает
  их назначение (тех/UX/конверсия/эвристики). Все они требуют логина и живого
  URL, публичного API нет — автоматом не прогнать.
- **VWO (heatmaps)** — заменяется бесплатной **Яндекс.Метрикой** (вебвизор, карта
  кликов/скроллинга, аналог rage-clicks).
- **PageSpeed Insights** — даст реальные метрики (LCP/CLS/TBF), но **нужен
  публичный URL** → сначала задеплоить (GitHub Pages при публичном репо / Netlify
  / Vercel), потом прогнать.
