# РегионДорСтрой — сайт

Лендинг дорожно-строительной компании ООО «РегионДорСтрой» (г. Орёл): дорожное
строительство, благоустройство, аренда спецтехники, поставка сыпучих материалов.

## Файлы

- `index.html` — production-сайт (standalone, без зависимостей кроме Google Fonts и Яндекс.Карты).
- `assets/logo-opt.png` — логотип компании (растровый бейдж, 226×300).
- `favicon.svg` — фавикон (жёлтый знак «Р»).
- `privacy.html` — политика конфиденциальности (152-ФЗ).
- `404.html` — страница ошибки (русская, в стиле сайта).
- `robots.txt`, `sitemap.xml` — для поисковиков.
- `.nojekyll` — отключает обработку Jekyll на GitHub Pages.
- `telegram-worker.js` — Cloudflare Worker для отправки заявок в Telegram.
- `AUDIT.md`, `AUDIT-PROD-2026-06-26.md` — экспертные аудиты сайта.

## Источник дизайна

Дизайн ведётся в **Claude Design**, проект `РегионДорСтрой — дизайн сайта`
(id `de7e6e79-40b1-439a-ad97-b17122492ed1`). `index.html` компилируется из него:
шаблоны развёрнуты в статику, интерактив перенесён на vanilla JS. Поверх дизайна
в `index.html` добавлен **production-слой** (SEO, формы, аналитика, доступность) —
он живёт в репозитории; при пересборке из Claude Design его нужно переносить заново.

## Деплой (GitHub Pages)

Репозиторий `Oreopaks/Site` → **Settings → Pages → Source = Deploy from a branch →
`main` / `(root)`**. Это **project page**, публичный URL — `https://oreopaks.github.io/Site/`
(с сегментом `/Site/`). Пуш в `main` = деплой. После — открыть сайт с `Ctrl+F5`
и прогнать PageSpeed Insights по публичному URL.

Все внутренние ссылки и ассеты — относительные (`assets/...`, `favicon.svg`,
`privacy.html`), поэтому работают под `/Site/`. Абсолютные URL (canonical, og:*,
JSON-LD, sitemap, robots) указывают на `https://oreopaks.github.io/Site/`.

### Custom domain (опционально)

Свой домен (напр. `regiondorstroy.ru`) солиднее и убирает сегмент `/Site/`:
добавить файл `CNAME` с доменом, настроить DNS на GitHub Pages, затем заменить
во всех абсолютных URL `https://oreopaks.github.io/Site/` → `https://<домен>/`
(в `index.html`, `robots.txt`, `sitemap.xml`) и `ALLOW_ORIGIN` воркера.

## ⚙️ Что настроить перед полноценным запуском

1. **Telegram-заявки** — задеплоить `telegram-worker.js` на Cloudflare (инструкция
   в шапке файла), вписать URL воркера в `index.html` → `CFG.tgEndpoint`. До этого
   форма при отправке предлагает связаться **напрямую** (WhatsApp / звонок) —
   ложный экран «успеха» не показывается, лиды не теряются.
2. **Реквизиты** — вписать ИНН/ОГРН/юр.адрес в `privacy.html` и подвал `index.html`
   (плейсхолдеры `[УКАЗАТЬ]`). Без них политика 152-ФЗ недействительна.
3. **Логотип** — `assets/logo-opt.png` содержит **устаревший телефон 46-90-81** и
   весит 91 КБ при рендере 84px. Нужен новый файл с актуальным номером (49-11-11),
   сжатый (pngquant/WebP, цель <15 КБ).
4. **Аналитика** — вставить номер счётчика Яндекс.Метрики в `index.html`
   (`var METRIKA_ID = 0;` → реальный id). Cookie-уведомление уже добавлено.
5. **Реальные фото** — заменить плейсхолдеры техники/объектов (блоки `.slot`).
6. **Проверить аккаунты** — убедиться, что `wa.me/79807691111` и
   `t.me/regiondorstroy` — реальные рабочие аккаунты, телефоны отвечают.
